# BTY Training Center — 진행률 트래커

**목적:** 매일 작업 후 "오늘 얼마나 끝났는지", "이만큼 했으면 다음은 뭘 해야 하는지"를 한눈에 판단.  
**세 가지 큰 프로젝트:** 서류작업 / 코드작업 / 배포작업.  
**코드작업 하위:** Center, Arena, Foundry(Foundry 내 동영상·애니메이션 제작 포함).

**코드·배포 진행률:** 서류는 문서 존재 여부로 판단할 수 있으나, **코드와 배포는 실제 저장소·인프라를 열어봐야** 채울 수 있다. **감사 2026-03-21:** `bty-app/` 기준으로 Center·Arena·Foundry·배포 항목을 대조해 `[x]`·요약 표를 반영했다. (항목 정의가 “전부 완비”인 경우는 미충족이면 `[ ]`로 둠 — 예: F5 12개 프로그램 전부·F6·F7 제작물, P5 모니터링 전용 구성.)

---

## 사용 방법

1. **작업을 완료하면** 해당 항목의 `[ ]`를 `[x]`로 바꾼다.
2. **완료한 항목의 점수만 합산**해서 아래 "현재 진행률" 표에 반영한다.
3. **"진행률이 얼마야?" 답변용** → 합산한 숫자를 **`docs/PROGRESS_SNAPSHOT.md`**에 갱신한다. (한 문서로 정확한 답 가능)
4. **오늘 한 작업**은 문서 맨 아래 "오늘 작업 로그"에 짧게 적는다.
5. **다음에 할 일**은 "미완료 항목" 또는 **`docs/ROADMAP_IMPLEMENTATION.md`** 순서대로 선택한다.

## 전체 가중치 (100% 기준)

| 큰 프로젝트 | 비중 | 만점 점수 | 설명 |
|-------------|------|-----------|------|
| 서류작업 | 20% | 100점 | 설계·정의·분류·포맷·가이드 문서 |
| 코드작업 | 60% | 300점 | Center + Arena + Foundry(콘텐츠·동영상·애니메이션 포함) |
| 배포작업 | 20% | 100점 | 인프라·배포·운영 |
| **합계** | 100% | **500점** | |

**전체 완성률(%)** = (서류 점수 + 코드 점수 + 배포 점수) / 500 × 100

**비율 검토:** 20 / 60 / 20은 **목표 가중치**로 두는 것이 맞다.  
- 서류 20%: 설계·정의가 코드/배포를 이끌므로 적절.  
- 코드 60%: 실제 제품 구현 비중이 가장 크므로 유지.  
- 배포 20%: MVP만 해도 로컬·스테이징·프로덕션·체크리스트가 필요하므로 유지.  
**현재 상태**는 “완료한 항목 점수”로만 계산하면 된다. 지금은 서류 일부만 완료되어 서류 비중이 실제로 더 크게 나오는 것이 정상이다.

## 4. 현재 진행률 요약

*아래 숫자를 매번 작업 후 **직접** 갱신한다. 코드·배포는 실제 구현/인프라를 확인한 뒤 채운다.*

| 큰 프로젝트 | 완료 점수 | 만점 | 진행률 |
|-------------|-----------|------|--------|
| 서류작업 | 45 | 100 | 45% |
| 코드작업 | | | |
| ├ Center | 60 | 60 | 100% |
| ├ Arena | 75 | 120 | 62.5% |
| └ Foundry | 65 | 120 | 54.2% |
| **코드 소계** | **200** | **300** | **66.7%** |
| 배포작업 | 80 | 100 | 80% |
| **전체** | **325** | **500** | **65%** |

**참고:** 위 수치는 **2026-03-21 `bty-app/` 감사** 기준. `[ ]`인 항목(A4~A6, A8, F5~F7, P5~P6)은 미충족 또는 미확인으로 두었다. 이후 구현 시 `[x]`로 바꾸고 합산을 다시 맞춘다.

### 코드 진행률 확인 방법

코드가 어디까지 구현됐는지 확인하려면:

1. **프로젝트 루트**에서 아래 항목에 해당하는 파일·API·UI가 있는지 확인한다.
2. **Center:** 로그인/세션, Arena·Foundry 진입 라우트/메뉴, 진도 표시, 대시보드 페이지.
3. **Arena:** 시나리오 데이터 로드, 선택지(Choice) UI, 선택 시 플래그 저장, Mirror 문구 표시, Foundry 추천 연결.
4. **Foundry:** 프로그램 목록·메타, 수강/완료 상태, Arena 완료 후 추천 로직, 프로그램 콘텐츠 페이지.
5. **배포:** `package.json` 스크립트, Docker/CI 설정, 스테이징·프로덕션 배포 설정 파일.

확인 후 해당 항목의 `[ ]`를 `[x]`로 바꾸고, 완료 점수를 합산해 위 "현재 진행률 요약" 표에 반영한다. (코드 저장소가 다른 경로에 있으면, 그 경로 기준으로 같은 방식으로 점검하면 된다.)

### MVP 목표 (Phase 1만 완료 시 대략)

- 서류: D1, D2, D3(1차 분류), D4(최소 10개 Core), D5 완료 → 약 70점 이상.
- 코드: Center C1~C5, Arena A1~A3·A5(선택), Foundry F1~F5·F5(최소 3개 콘텐츠) → 약 180점 이상.
- 배포: P1, P2, P4(또는 preview), P6 → 약 65점 이상.
- **MVP 전체** ≈ (70 + 180 + 65) / 500 ≈ **63%** 수준이면 Phase 1 배포 가능으로 가정.
