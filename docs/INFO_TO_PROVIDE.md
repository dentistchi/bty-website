# AI에게 알려주면 좋은 정보 (체크리스트)

버그 수정·기능 수정 요청 시 아래 정보를 알려주면, 그에 맞게 수정 후 배포까지 진행합니다.

---

## 1. 어떤 문제/요청인지

- **에러인 경우**: 화면에 보이는 에러 문구 (예: `UNAUTHENTICATED`, `401`, `Failed to fetch`)
- **동작 요청인 경우**: 하고 싶은 것 한 줄 (예: "로그인 후 대시보드로 바로 가게 해줘")

---

## 2. 어디서 발생하는지

- **URL**: 지금 보고 있는 주소 (예: `https://bty-website.ywamer2022.workers.dev/en/bty/leaderboard`)
- **언어**: en(메인) / ko(보조) 중 어떤 사이트에서인지
- **화면/기능**: Dashboard / Arena / Leaderboard / 로그인 / 로그아웃 등

---

## 3. 재현 방법 (가능하면)

- 예: "로그인 → Dashboard → Leaderboard 클릭하면 에러"
- 예: "한 번 로그아웃한 다음 다시 로그인하면 쿠키가 /ko/... 로 바뀜"

---

## 4. 스크린샷/캡처 (있으면 좋음)

- **에러일 때**: 에러 메시지가 보이는 화면
- **개발자 도구**: Console 탭(빨간 에러), Network 탭(실패한 요청 401 등), Application 탭(쿠키 Path/값)

---

## 5. 이미 시도한 것 (있다면)

- 예: "쿠키 다 지우고 다시 로그인해도 같음"
- 예: "다른 브라우저에서는 됨"

---

## 요약 템플릿 (복사해서 쓰기 좋게)

```
- 문제: (한 줄 요약)
- URL: 
- 화면: (Dashboard / Arena / Leaderboard / 로그인 등)
- 재현: (단계)
- 스크린샷: (붙이거나 "Console/Network/Application 캡처함")
```

이 정보를 주시면, 수정 후 **중간 확인 없이 커밋·푸시(배포)** 까지 진행합니다.
