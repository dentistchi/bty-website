# 진행률 한눈에 (Center / Arena / Foundry / 전체)

**목적:** "코드 진행 중에 진행률이 얼마야?"에 **한 문서로 답**할 수 있게 한다.  
**갱신:** `docs/PROGRESS_TRACKER.md`에서 완료 항목([x])·점수 합산 후, 이 문서의 숫자만 갱신한다.

**주의 — 두 가지 %:**  
- **제품 기능 완료율(Phase 기준)**은 **`docs/MVP_AND_COMPLETION_INDICATORS.md`** (§3.1)를 쓴다. **이 문서의 %는 500점 모델**(`PROGRESS_TRACKER.md`) 합산이다.  
- 아래 숫자가 **코드 0%**로 보이면, 트래커가 **아직 `[x]` 감사 전**일 수 있다. **`PROGRESS_TRACKER.md`를 코드베이스와 대조**한 뒤 갱신한다.

## 마지막 갱신

| 항목 | 값 |

| **갱신 기준** | `PROGRESS_TRACKER.md` 완료 항목·점수 합산 (`bty-app/` 감사) |

## 1. 코드 — Center / Arena / Foundry

| 영역 | 완료 점수 | 만점 | 진행률 |
|------|-----------|------|--------|
| **Center** | 60 | 60 | 100% |
| **Arena** | 75 | 120 | 62.5% |
| **Foundry** | 65 | 120 | 54.2% |
| **코드 합계** | 200 | 300 | 66.7% |

**계산:** 각 영역 진행률 = (완료 점수 / 만점) × 100. 코드 합계 = (Center + Arena + Foundry 점수 합) / 300 × 100.

---

## 2. 서류·배포·전체

| 구분 | 완료 점수 | 만점 | 진행률 |
|------|-----------|------|--------|
| **서류작업** | 45 | 100 | 45% |
| **코드작업** | 200 | 300 | 66.7% |
| **배포작업** | 80 | 100 | 80% |
| **전체** | **325** | **500** | **65%** |

**전체 진행률** = (서류 + 코드 + 배포 완료 점수) / 500 × 100.

---

## 3. "진행률이 얼마야?" 답변 템플릿

갱신 후 아래처럼 답하면 된다.

- **Center:** "Center는 60/60점, **100%** (트래커 항목 기준)."
- **Arena:** "Arena는 75/120점, **62.5%**."
- **Foundry:** "Foundry는 65/120점, **54.2%**."
- **전체(500점 모델):** "전체는 325/500점, **65%**야. 서류 45%, 코드 200점, 배포 80점."
- **제품 Phase %가 따로 필요하면:** `MVP_AND_COMPLETION_INDICATORS.md` §3.1 참고.

---

## 4. 갱신 방법

1. `docs/PROGRESS_TRACKER.md`를 연다.
2. 코드·배포 완료된 항목이 있으면 해당 행의 `[ ]`를 `[x]`로 바꾼다.
3. **Center:** C1~C6 중 [x]인 항목의 점수를 더해 "완료 점수"를 구한다. → 이 문서 "Center 완료 점수·진행률"에 기입.
4. **Arena:** A1~A9 동일. → 이 문서 "Arena 완료 점수·진행률"에 기입.
5. **Foundry:** F1~F7 동일. (F6 부분 완료 시 12개 중 n개 완료면 n/12×24점.) → 이 문서 "Foundry 완료 점수·진행률"에 기입.
6. **배포:** P1~P6 동일. → 이 문서 "배포 완료 점수·진행률"에 기입.
7. 이 문서 상단 "마지막 갱신" 날짜를 오늘로 바꾼다.

코드베이스를 확인해서 "이 기능 구현돼 있음"을 판단한 뒤 PROGRESS_TRACKER에 [x]를 넣고, 그 결과를 이 문서에 반영하면 **정확한 진행률**이 나온다.
