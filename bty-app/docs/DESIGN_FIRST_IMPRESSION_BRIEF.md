# 첫인상 디자인 브리프 — "전문가 손길"이 느껴지게

**목표**: 보는 사람이 "그냥 홈페이지"가 아니라 **"우와, 이거 장난 아닌데. 전문가가 만든 거 같다. 매일 쓰고 싶은 사이트다"** 라는 반응을 하도록, **첫인상**을 올린다.

**현재**: Arena 레이아웃(크림·보라 톤, 카드, 네비)은 적용됐지만 **밋밋하다**는 피드백. 한 단계 위로 끌어올리기.

---

## 1. 전문가 느낌을 주는 핵심 요소 (우선 적용)

| 요소 | 밋밋한 상태 | 전문가 느낌으로 바꾸는 법 |
|------|-------------|---------------------------|
| **타이포그래피** | 시스템 폰트, 균일한 크기 | **한 가지 포인트 폰트** 도입 (제목·로고만). 예: Pretendard Bold, Noto Sans KR (제목 700), 본문은 기존 유지. 제목은 **크기 대비**로 크게(2rem~2.5rem), 자간·줄간격 정돈. |
| **계층·여백** | 카드가 비슷해 보임 | **한 페이지에 "한 가지 주인공"**. 랜딩/대시보드 상단에 **히어로 블록 1개** (한 줄 문구 + CTA 또는 오늘의 한 줄). 나머지는 카드로 **그룹화**하고, 그룹 간 **세로 여백**을 넉넉히(32px~48px). |
| **색·대비** | 크림·보라만 쓰면 단조로움 | **악센트 1색**을 정해서 **클릭·강조·진행**에만 씀. 예: 보라 말고 **딥 틸/세이지** 또는 **한 가지 코랄**을 버튼·링크·프로그레스 바에만. 배경은 크림 유지해 **대비**로 "선택된 것"이 확 들어오게. |
| **미세한 움직임** | 정적 | **호버·포커스**에서만: 버튼/카드 **scale(1.02)** 또는 **밝기 5% 올리기**, **transition 0.2s**. 로딩 시 **카드가 순서대로 살짝 페이드인**(0.05s 간격). 과하지 않게. |
| **진입점(랜딩/대시보드 첫 화면)** | 텍스트 나열 | **한 문장**만 크게: "오늘도 한 걸음, Arena에서." 같은 문구를 **히어로**로 두고, 그 아래에만 카드/CTA. **여백**을 많이 줘서 "정돈된 전문 서비스" 느낌. |

---

## 2. "매일 쓰고 싶다"를 위한 디테일

| 항목 | 제안 |
|------|------|
| **로고·브랜딩** | "BTY Arena"를 **작은 로고 타입**으로 (폰트·두께·색 고정). 헤더 왼쪽에만 두고, 클릭 시 메인/대시보드로. |
| **네비** | 활성 탭만 **배경+글자색**으로 강조. 나머지는 **회색 톤**으로 두고, 호버 시에만 진하게. "지금 여기"가 한눈에 들어오게. |
| **카드** | 그림자만으로 구분 말고, **호버 시 살짝 들어올림**(transform translateY(-2px) + shadow 키우기). "클릭 가능·상호작용하는 UI" 느낌. |
| **빈 상태·로딩** | "데이터 없음"일 때 **일러·아이콘 + 한 줄 문구** (예: "아직 기록이 없어요. 첫 시나리오를 시작해 보세요."). 스피너 말고 **스켈레톤** 또는 **카드 형태 로딩**으로 일관감. |
| **일관된 톤** | 버튼은 **한 가지 스타일**(예: 둥근 12px, 채움형). 보조 액션은 **아웃라인만**. "어디를 눌러야 할지" 직관적으로. |

---

## 3. 적용 순서 제안 (최소 비용으로 첫인상 올리기)

1. **히어로 1블록** — 대시보드/랜딩 상단에 **한 문장 + 넉넉한 여백**. (이미 문구는 있으면, **크기·위치**만 올림.)
2. **포인트 폰트 1종** — 제목·로고에만 웹폰트 적용 (Pretendard, Noto Sans KR, 또는 영문만 Inter/Satoshi).
3. **악센트 색 1개** — 버튼·활성 탭·프로그레스에만 적용. 나머지 크림/보라는 유지.
4. **호버·포커스** — 카드·버튼에 transition + 살짝 scale/밝기 변화.
5. **카드 그림자·호버** — 기본 shadow, 호버 시 살짝 키움 + translateY.

---

## 4. 복사용 프롬프트 (진행 에이전트용)

### A. 첫인상 히어로 + 여백 (1단계)

```
bty-app 대시보드와 Arena 진입 첫 화면의 "첫인상"을 올려줘.

- 대시보드 상단: 기존 "오늘도 한 걸음, Arena에서." 문구를 **히어로 블록**으로. 폰트 크기 1.75rem~2rem, font-weight 700, 상하 여백 32px~48px. 그 아래 카드 영역과 시각적으로 분리.
- Arena(/bty-arena) 첫 화면도 동일: 한 문장 히어로 + 넉넉한 여백 후 본문.
- 전체적으로 카드 그룹 간 세로 gap 24px~32px, max-width 유지.
```

### B. 포인트 폰트 + 악센트 색 (2단계)

```
bty-app에 "전문가 느낌" 타이포·색을 적용해줘.

- 제목·로고: 웹폰트 1종 적용 (예: Pretendard Variable 또는 Noto Sans KR). 제목만 font-weight 700~800, 크기 1.5rem 이상.
- 악센트 색 1개: CSS 변수 --arena-accent (예: #5B4B8A 또는 세이지 #6B7B5A). 버튼(primary), 활성 네비 탭, 프로그레스 바에만 사용. 배경 크림은 유지해 대비 살림.
- 본문은 기존 폰트 유지, line-height 1.6.
```

### C. 호버·카드 인터랙션 (3단계)

```
bty-app 카드·버튼에 미세한 인터랙션을 넣어줘.

- ProgressCard·클릭 가능 카드: hover 시 transform translateY(-2px), box-shadow 약간 강화, transition 0.2s ease.
- 버튼: hover 시 brightness(1.05) 또는 scale(1.02), transition 0.15s.
- 포커스 링: focus-visible 시 outline 대신 box-shadow로 링 (접근성 유지).
```

---

## 5. 한 줄 요약

- **"밋밋함 탈출"** = 히어로 1블록 + 포인트 폰트 1종 + 악센트 색 1개 + 호버/카드 미세 인터랙션.
- **첫인상** = 진입 화면에서 **한 문장이 크게**, **여백이 넉넉히**, **색·타이포가 정돈**되어 있으면 "전문가 손길"이 느껴짐.
- 적용 순서: **§4 A → B → C** (히어로·여백 → 폰트·악센트 → 호버·인터랙션).

---
